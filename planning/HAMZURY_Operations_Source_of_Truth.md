## 5. TASK ASSIGNMENT MODEL

Tasks are not permanently assigned to one person. The model is flexible:

1. CSO receives lead → creates task → assigns to department
2. Department Lead assigns task to available staff or handles it directly
3. Task has: expected delivery date, department tag, status, assigned staff
4. CEO reviews all tasks weekly at hub meeting
5. Finance tracks task completion for commission calculation

Task statuses in system: pending, in_progress, completed, cancelled
Department tags: bizdoc, systemise, skills, hr, cso, finance, founder, media

There is no "my permanent clients" model. Staff work on tasks as assigned.

## 9. DASHBOARD GOVERNANCE RULES

### Who Sees What
- **Founder:** All dashboards (cross-visibility across every unit)
- **CEO:** CEO dashboard + all department dashboards via cross-department task view
- **CSO:** CSO dashboard only — sees all leads, assigns to departments, tracks pipeline
- **Finance:** Finance dashboard only — sees revenue, expenses, commissions, subscriptions
- **Department Lead:** Their department dashboard only — sees their queue, projects, staff
- **Staff:** StaffWorkspace only — sees their assigned tasks, attendance, personal KPIs
- **Client:** ClientDashboard only — sees their project progress, deliverables, chat

### Dashboard Must-Haves
- Every dashboard must use tRPC-authenticated data (no mock fallbacks in production)
- Every dashboard must respect the role boundary (no cross-role data leakage)
- Client dashboard must verify client session server-side (not localStorage-only)
- No hardcoded credentials anywhere in client code
- No dashboard should expose pricing, commission rates, or internal KPIs to external users

### Dashboard Must-Nots
- No dashboard should allow staff to see other staff's commission details
- No dashboard should expose client contact details to non-CSO/non-department-lead roles
- No dashboard should allow task reassignment without CEO/Department Lead authority
- Client dashboard should not show internal staff names or assignment details
